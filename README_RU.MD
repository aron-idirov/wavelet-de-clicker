# Wavelet-Based Audio Declicker

![Интерфейс приложения](images/gui.png)

Прототип аудио-де-кликера для обнаружения и подавления импульсных помех в аудиосигналах с использованием дискретного вейвлет-преобразования (DWT).

Проект разработан в рамках выпускной квалификационной работы и предназначен как исследовательский прототип для экспериментов с вейвлетной обработкой аудиосигналов.

## Описание

Импульсные помехи в аудиозаписях обычно воспринимаются как щелчки, треск, хлопки или короткие резкие артефакты. Такие дефекты часто локализованы во времени и могут иметь широкий спектральный состав, поэтому их сложно удалять простыми методами фильтрации.

В проекте реализован настраиваемый алгоритм де-клика, основанный на:

- дискретном вейвлет-преобразовании;
- событийной детекции импульсных помех;
- локальной обработке детализирующих вейвлет-коэффициентов;
- многополосном подавлении в вейвлетной области;
- визуальном анализе исходного и обработанного сигнала.

Приложение включает графический интерфейс для загрузки аудиофайлов, настройки параметров обработки, запуска алгоритма, просмотра найденных событий и сохранения результата.

## Возможности

- Загрузка и обработка аудиофайлов.
- Поддержка моно- и стереосигналов.
- Детектирование импульсных помех на основе вейвлет-полос.
- Локальное подавление найденных щелчков.
- Выбор семейства вейвлетов и уровня DWT-разложения.
- Настройка диапазона обрабатываемых вейвлет-полос.
- Настройка параметров детекции и подавления.
- Поддержка циклических сдвигов для более устойчивой DWT-обработки.
- Визуализация:
  - исходного сигнала;
  - обработанного сигнала;
  - спектрограммы разности;
  - отдельных найденных кликов.
- Сохранение обработанного аудиосигнала в файл.

## Используемые технологии

Проект написан на Python и использует следующие библиотеки:

- `NumPy` — численные вычисления и работа с массивами;
- `PyWavelets` — дискретное вейвлет-преобразование и обратная реконструкция;
- `SoundFile` — чтение и запись аудиофайлов;
- `SciPy` — вспомогательная обработка сигналов и построение спектрограмм;
- `Matplotlib` — визуализация формы волны и спектрограмм;
- `Tkinter` — графический интерфейс пользователя.

## Алгоритм

Алгоритм использует событийную многополосную обработку в области вейвлет-коэффициентов.

Общая схема работы:

1. Загружается входной аудиосигнал.
2. Для стереосигнала формируется связанный сигнал детекции по обоим каналам.
3. Выполняется DWT-разложение.
4. Из выбранных detail-полос восстанавливается сигнал для детекции.
5. Импульсные события обнаруживаются с помощью робастного MAD-порога.
6. Близкие срабатывания объединяются в отдельные события.
7. Выполняется DWT-разложение исходного сигнала.
8. Для каждого найденного события и выбранной detail-полосы:
   - оценивается локальная медиана и sigma;
   - рассчитывается сила выброса;
   - автоматически подбирается сила подавления;
   - ограничиваются аномальные вейвлет-коэффициенты.
9. Сигнал восстанавливается с помощью обратного DWT.
10. Результат визуализируется и может быть сохранён в аудиофайл.

Алгоритм не применяет глобальную фильтрацию ко всему сигналу. Вместо этого он изменяет только локальные области вокруг найденных событий и только в выбранных вейвлет-полосах.

## Основные параметры GUI

- **Wavelet** — семейство и тип вейвлета для DWT-разложения.
- **Border mode** — режим продолжения сигнала на границах.
- **DWT level** — количество уровней вейвлет-разложения.
- **Band high / Band low** — диапазон detail-полос, используемых для детекции и обработки.
- **Detect MAD k** — коэффициент порога детектирования.
- **Event (ms)** — предполагаемая длительность импульсного события.
- **Context (ms)** — контекстное окно для оценки локальной статистики.
- **Sigma trim q** — параметр усечения выбросов при расчёте sigma.
- **k_min / k_max** — диапазон силы ограничения коэффициентов.
- **Reaction** — чувствительность реакции алгоритма на силу выброса.
- **Cycle Spins** — количество циклических сдвигов для уменьшения зависимости результата от сетки DWT.
